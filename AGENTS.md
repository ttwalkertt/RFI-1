# AGENTS.md

Before working on this repository, read:

1. `docs/framework-task-operating-model.md`
2. `TASKS.md`
3. the active task ticket under `tasks/`

Use the task ticket for task-specific objectives, boundaries, invariants, acceptance criteria, and evidence requirements.

Use `docs/framework-task-operating-model.md` for project-wide expectations about architectural milestones, scope discipline, verification, review packages, and the required Architectural Status Summary.

Do not infer task completion without required evidence.

Task-specific instructions override project-wide guidance only when they explicitly conflict.
